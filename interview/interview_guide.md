## System Design Interview guide

### How to prepare for a System Design Interview?
1. Understand what's expected
2. Refresh your fundamentals
3. Familiarize yourself with the basic building blocks
4. Work backwards from common problems and iterate

### What framework should you follow to present your design?
1. Requirements
2. Core Entities
3. API or interface
4. Data Flow
5. High-level design --> Back to step 1 (Primary goal: Functional requirements)
6. Deep Dives --> Back to step 1 (Primary goal: Non-functional requirements)

### How are system design interviews evaluated?
1. Problem solving: Identify and prioritize the core challenges
2. Solution design: Create scalable architectures with balanced trade-offs
3. Technical excellence: Demonstrate deep knowledge and expertise
4. Communication: Clearly explain complex concepts to stakeholders

### The Fundamentals
1. Storage
2. Scalability
3. Networking
4. Latency, Throughput and Performance
5. Fault tolerance and Redundancy
6. CAP Theorem
